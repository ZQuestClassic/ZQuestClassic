1 // Scripting arrays are declared by static array registrars that run at process start. For example:
2 //
3 // static ArrayRegistrar GAMEBOTTLEST_registrar(GAMEBOTTLEST, []{
4 // static ScriptingArray_ObjectMemberCArray<gamedata, &gamedata::bottleSlots> impl;
5 // impl.setMul10000(true);
6 // return &impl;
7 // }());
8 //
9 // The above configures the GAMEBOTTLEST variable to map to the bottleSlots field of gamedata.
10 //
11 // static ArrayRegistrar IDATAWPNINITD_registrar(IDATAWPNINITD, []{
12 // static ScriptingArray_ObjectSubMemberCArray<itemdata, &itemdata::weap_data, &weapon_data::initd> impl;
13 // impl.compatSetDefaultValue(-10000);
14 // impl.setMul10000(false);
15 // return &impl;
16 // }());
17 //
18 // The above configures the IDATAWPNINITD variable to map to the initd field of weapon_data, inside the weap_data member of itemdata.
19 //
20 // Steps for defining new scripting arrays:
21 //
22 // 1. Locate the scripting/types files that corresponds to the bindings .zh file for the ZASM variable.
23 //
24 // 2. Copy the "static ArrayRegistrar" boilerplate from somewhere else, following the naming convention
25 // of ZASM_VAR_registrar.
26 //
27 // 3. Determine which ScriptingArray interface fits best. These are all the options:
28 //
29 // ScriptingArray_GlobalCArray
30 // ScriptingArray_GlobalComputed
31 // ScriptingArray_ObjectMemberCArray
32 // ScriptingArray_ObjectSubMemberCArray
33 // ScriptingArray_ObjectMemberBitwiseFlags
34 // ScriptingArray_ObjectSubMemberBitwiseFlags
35 // ScriptingArray_ObjectMemberBitstring
36 // ScriptingArray_ObjectMemberContainer
37 // ScriptingArray_ObjectComputed
38 //
39 // The "Global" interfaces pass the ref variable as an int. Many ZASM arrays are "global" in that they
40 // do not need to resolve a context object (like `Game->FFRules[]`).
41 //
42 // The "Object" interfaces take a type parameter for the scripting object that ref resolves for, and
43 // handles getting the object and checking for its existence.
44 //
45 // Some interfaces are very simple to use, such as: GlobalCArray, ObjectMemberCArray,
46 // ObjectMemberBitwiseFlags, ObjectMemberBitstring, ObjectMemberContainer. All of these simply require a pointer to the struct
47 // field as a type parameter (see `&gamedata::bottleSlots` in the first above example).
48 //
49 // Some are similarly easy to use, such as: ObjectSubMemberCArray, ObjectSubMemberBitwiseFlags.
50 // These require a pointer to the struct field, and a pointer to a field on THAT struct as well.
51 // (see `&itemdata::weap_data` and `&weapon_data::initd` in the second above example)
52 //
53 // The GlobalComputed and ObjectComputed interfaces are more involved. They take 3 lambdas each: one for
54 // calculating the size, one for the getter, and one for the setter. For getters and setters, the index is
55 // validated based on the size lambda, so no need to further validate in those lambas. For ObjectComputed,
56 // the lambdas run only if the context object could be resolved, so no need to check it is valid. Lastly,
57 // the values given to the getter/setter/setValueTransform lambdas have already been modified based on the
58 // value of `impl.setMul10000(bool)` (see next section).
59 //
60 // Sometimes the "Object" interfaces don't work for a given array, for example if the ref integer value is
61 // needed; in these cases it's fine to use "GlobalComputed". Just be sure to validate the ref is valid in
62 // each lambda.
63 //
64 // 4. Configure the array.
65 //
66 // a) Configure the integer type (false for "long", true for "fixed-point int"): impl.setMul10000(false)
67 //
68 // b) If the value is bounded or validated, use `impl.setValueTransform(...)` For example, to vbound between
69 // 0 and 255: `impl.setValueTransform(transforms::vboundByte)`. There's also `transforms::vbound<low, min>`
70 // to use any bounds. To validate instead (disallowing values outside a range), use
71 // `transforms::validate<low, max>`. Lastly, you can use any arbitary lambda function: return std::nullopt
72 // to reject a value, and return an integer otherwise (optionally modifying it).
73 //
74 // c) Configure `impl.setSideEffect(...)` to run some code after an array value is successfully set. Useful for an
75 // array that otherwise fits the use case of the simpler interfaces (no need to reach for the more general
76 // Global interface).
77 //
78 // d) If read-only: `impl.setReadOnly()`
