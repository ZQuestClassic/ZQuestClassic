41 /*
42 z3-style scrolling is implemented via "Regions".
43
44 Regions:
45
46 - are a NxM set of screens within a single map
47 - even 1x1 (one screen) areas are regions. A region containing a single screen is called
48 a "non-scrolling region", otherwise it is a "scrolling region"
49 - must be a rectangle
50 - are defined in the Region Editor by assigning each screen an id, but other than
51 grouping screens into the same region
52 this id currently has no purpose in-engine
53 - can border other regions or single screens, and supports scrolling between them
54 (including side warps)
55 - when moving around a scrolling region, by default the viewport is updated to keep
56 the player drawn in the center of the screen. When moving close to the region edge,
57 the viewport bounds to the edges. This behavior is modified by `viewport_mode`, which
58 can be modified by scripts via `Viewport->`
59 - the top-left screen is loaded as `cur_screen` / `origin_scr`
60 - `hero_screen` / `hero_scr` is the screen where the hero currently is, and updates as the
61 player moves around.
62 */
