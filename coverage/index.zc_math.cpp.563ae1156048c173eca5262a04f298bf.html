8 // Math that produces bit-identical results on every platform and compiler,
9 // accurate enough (error < 1e-6) that swapping it for the libm equivalents is
10 // invisible at the script layer, where values truncate to 1e-4. This is the
11 // only trig used for gameplay since replay version 59 - whether or not a
12 // replay is active - so recording a replay can never change behavior.
13 //
14 // Angles are reduced to integer "turns" (TURN_BITS per full circle), then a
15 // Q30 quarter-wave table is interpolated with int64 math. The only floating
16 // point operations are the fmod/multiply/round on input (each individually
17 // exact or correctly rounded per IEEE 754, so deterministic even under
18 // -ffp-model=fast) and the final scale by a power of two (exact).
19 //
20 // Determinism invariant: every multiply that sits next to an add must be a
21 // power-of-two scale (exact), so FMA contraction under -ffp-model=fast cannot
22 // change results. Keep that property when editing.

24 // History: before version 59 (and only while a replay was active), trig used a
25 // 32-entry Q15 table with results rounded to 3 decimals. That error (~1e-3)

28 // See https://discord.com/channels/876899628556091432/1509803675605008384
